# ขั้นตอนการ Export Database จาก phpMyAdmin พร้อมข้อมูลครบถ้วน

หากเกิดปัญหาข้อมูลไม่มา หรือเกิด Error เกี่ยวกับ Foreign Key Constraint (เช่น `errno: 150`) ตอนที่นำไฟล์ Database ไป Import ลงอีกเครื่องหนึ่ง สามารถแก้ไขได้โดยการตั้งค่าการ Export ใหม่ดังนี้:

## วิธีการ Export อย่างละเอียด

1. เข้าไปที่ **phpMyAdmin** แล้วเลือก Database ของคุณ (เช่น `it_service`)
2. ไปที่แท็บ **Export (ส่งออก)**
3. ในส่วนของ **Export method (วิธีการส่งออก)** ให้เปลี่ยนจาก **Quick (ด่วน)** เป็น **Custom (กำหนดเอง)**
4. ในส่วนของ **Tables (ตาราง)** ให้ตรวจสอบให้แน่ใจว่าเลือกทั้ง **Structure (โครงสร้าง)** และ **Data (ข้อมูล)** ครบทุกตารางแล้ว
5. เลื่อนลงมาหาหัวข้อ **Format-specific options (ตัวเลือกเฉพาะรูปแบบ)**
6. **สำคัญมาก:** ให้ติ๊กเลือกช่อง **"Disable foreign key checks" (ปิดการตรวจสอบ Foreign Key)** 
   - *คำอธิบาย: การติ๊กช่องนี้จะเพิ่มคำสั่ง `SET FOREIGN_KEY_CHECKS=0;` ไว้บนสุดของไฟล์ SQL ทำให้ตอน Import ไปอีกเครื่อง มันจะไม่ล็อกลำดับการสร้างและแทรกข้อมูลของตาราง และไม่เกิด Error ขัดจังหวะการลงข้อมูล*
7. ในส่วน **Object creation options (ตัวเลือกการสร้างออบเจ็กต์)** แนะนำให้ติ๊กเลือก **Add DROP TABLE / VIEW / PROCEDURE / FUNCTION / EVENT / TRIGGER statement** ไว้ด้วย
   - *คำอธิบาย: เพื่อให้ระบบลบตารางเก่าทิ้งให้เกลี้ยงก่อนสร้างใหม่ ป้องกันปัญหาตารางซ้ำซ้อนหรือโครงสร้างเก่าตีกัน*
8. เลื่อนลงมาด้านล่างสุด แล้วกดปุ่ม **Export (ส่งออก)**

เมื่อได้ไฟล์ `.sql` ใหม่มาแล้ว ให้นำไฟล์นี้ไป Import ลงในอีกเครื่อง ฐานข้อมูลของคุณจะมาครบถ้วนตามปกติครับ
